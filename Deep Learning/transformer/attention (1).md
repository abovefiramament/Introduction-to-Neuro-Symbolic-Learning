attention其关键在于将q,k,v的值经过各自w的矩阵积获得一个各自的功能位置与自身含义的叠加，

X × Wq = Q
（输入序列 × 投影权重 = 查询矩阵）
左边矩阵 X 的意义
• 行：一个 token（一个词 / 一个位置）
• 列：原来的特征维度
右边矩阵 Wq 的意义
• 列：一个 “新空间里的特征方向”
• 行：必须和 X 的列数一样（为了能做点积）

最后算出查询向量

结果矩阵 Q：L × dₖ
含义：每一行 = 这个 token 的查询向量

再对QK做矩阵积过滤评估相似度

S [i] [j] = 第 i 个位置 的特征・第 j 个位置 的特征

S = Q × Kᵀ
• 左矩阵 Q：L × dₖ
含义：每一行 = 查询方（我要去匹配谁）
• 右矩阵 Kᵀ：dₖ × L
含义：每一列 = 被匹配方（整个序列里的所有键）
• 结果矩阵 S：L × L
含义：
◦ 第 i 行第 j 列 = 第 i 个 token 对第 j 个 token 的匹配分数
◦ 整行 i = 第 i 个位置看遍全序列的得分

再归一化分配重要度，最后对v加权输出

Output = A × V
• 左矩阵 A：L × L
含义：每一行 = 第 i 个 token 的注意力权重（该听谁的）
• 右矩阵 V：L × dₖ
含义：每一行 = 要被加权的信息（值向量）
• 结果矩阵 Output：L × dₖ
含义：
◦ 第 i 行 = 第 i 个 token 融合全序列所有信息后的新特征

也就是将查询矩阵的所有位置叠加了对于我们需要了解的对key所有位置分配的权重后，最后对特征进行加权运算，将原矩阵特征叠加了查询特征，再叠加分配的注意力权重，最后解出

k改权重，v改特征，对于相似度以及特征突出度进行了匹配运算

Q-K-V 这套组合，等于给每一个 token 配了一个：
Q 是当前token的查询视角（需求）
K 是所有token的索引（匹配标准）
V 是所有token的真实内容（被提取的信息）
三者一起，让每个 token 都能从全序列里「精准、自动、按需」提取自己需要的信息。

多头注意力就是给予跟多的q-k-v映射方式，学会更多关系

事实上，所谓的多头注意力机制其实就是将原始的输入序列进行多组的自注意力处理；然后再将每一组得到的自注意力结果拼接起来，再通过一个线性层进行处理，得到最终的输出。我们用公式可以表示为：

MultiHead(Q,K,V)=Concat(head1,...,headh)WOwhere headi=Attention(QWiQ,KWiK,VWiV)

Layer Norm 的完整公式为：

```
LN(xi)=γ⊙ ((xi−μi)/根号(σi^2+ϵ))+β
```

其中：

- xi是第 i 个样本在该层的特征向量
- μi,σi是该样本自身特征的均值和标准差
- γ,β是该层所有样本共享的可学习参数
- ϵ是数值稳定项
