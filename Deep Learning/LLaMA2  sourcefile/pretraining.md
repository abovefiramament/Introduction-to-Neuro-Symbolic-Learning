# 大语言模型预训练（Pre-training）核心学习笔记

（与 SFT 严格区分：预训练是「通用语言能力打底」，SFT 是「对话 / 任务能力对齐」）

------

## 一、预训练在 LLM 全流程中的核心定位

预训练是**大语言模型的第一阶段**，目标是让模型在海量通用文本语料上学习「通用语言规律」（语法、语义、逻辑、知识），为后续的 SFT（有监督微调）、RLHF（人类反馈强化学习）打下坚实的基础。

### 2. 与 SFT 的核心区别（必须分开的原因）

|     维度     |             预训练（Pre-training）             |                      SFT（有监督微调）                      |
| :----------: | :--------------------------------------------: | :---------------------------------------------------------: |
| **核心目标** |  学习通用语言能力：语法、语义、逻辑、世界知识  | 对齐任务 / 对话格式：学会根据 user 输入生成 assistant 回复  |
| **数据类型** |    海量通用单条文本（维基百科、书籍、网页）    |               小规模结构化多轮对话 / 任务样本               |

| **损失计算** |   对**所有 token** 计算损失（因果语言建模）    | 仅对**assistant 回复部分**计算损失（忽略 user/system 输入） |

## 二、预训练的核心任务：因果语言建模（CLM, Causal Language Modeling）

预训练的唯一核心任务是**因果语言建模**：给定文本的前 t 个 token，预测第 t+1 个 token，这是 LLaMA、GPT 等所有自回归模型的基础。

### 1. 数学定义

对于输入文本序列 x=[x1,x2,...,xT]，模型的目标是最大化以下对数似然：

L=−T1∑t=1TlogP(xt∣x1,x2,...,xt−1)

### 2. 样本构造逻辑（对应 `PretrainDataset`）

- **输入 \**X\****：[x1,x2,...,xT−1]（去掉最后一个 token）
- **标签 \**Y\****：[x2,x3,...,xT]（去掉第一个 token，即「右移」）
- **训练逻辑**：模型输入 X，预测 Y，也就是 “用 x1 预测 x2，用 x1,x2 预测 x3，以此类推”。

### 3. 因果掩码（Causal Mask）

这是因果语言建模的关键：在注意力层中加入**下三角掩码**，防止模型在预测第 t 个 token 时看到 t 之后的 token（“未来信息”），确保模型只能根据上文预测下文。

------

## 三、预训练的核心数据准备

预训练的效果 90% 取决于数据质量，数据准备是预训练中最耗时、最关键的环节。

### 1. 数据来源（通用、海量、多样化）

预训练需要**覆盖广泛领域的海量通用文本**，常见来源：

- 维基百科（中文 / 英文）：高质量结构化知识
- CommonCrawl：海量网页文本
- 书籍语料库：长文本、逻辑连贯
- 开源代码语料：代码逻辑能力（可选）

### 2. 数据清洗（核心步骤，决定模型下限）

原始语料中存在大量脏数据，必须经过严格清洗：

1. **去重**：删除完全重复或高度相似的文本，避免模型过拟合。

2. 过滤低质量文本

   ：

   - 过滤短文本（<10 个 token）
   - 过滤乱码、广告、无意义文本
   - 过滤非目标语言文本（比如只保留中文）

   

3. **去敏感内容**：过滤暴力、色情、政治敏感内容，确保模型安全。

4. 标准化

   ：

   - Unicode 规范化（NFKC，对应 Tokenizer 训练时的 `NFKC()`）
   - 全角半角转换
   - 多余空格归一化

   

### 3. 数据预处理（对应 `PretrainDataset`）

清洗后的文本需要转换成模型可训练的样本：

1. **分词**：用预训练好的 Tokenizer 把文本转成 Token ID。
2. **截断**：如果文本太长，截断到 `max_seq_len`（比如 512、2048）。
3. **填充**：如果文本太短，用 `pad_token_id` 填充到 `max_seq_len`，确保所有样本长度一致。
4. **构造自回归样本**：生成 X（输入）、Y（标签，右移）、`loss_mask`（填充部分设为 0，忽略损失）。

## 四、预训练的模型架构与损失函数

### 1. 模型架构：因果 Transformer 解码器

预训练使用的是**仅解码器的 Transformer 架构**（Decoder-only Transformer），也就是 LLaMA2 的结构：

- 核心模块

  ：

  1. 词嵌入层（Embedding Layer）
  2. 多层 Transformer 解码器层（每层包含：RMSNorm、分组查询注意力 GQA、前馈网络 FFN、残差连接）
  3. 最终 RMSNorm + 线性输出层（LM Head）

  

- 关键特性

  ：

  - 因果掩码（下三角掩码），防止看到未来信息
  - 分组查询注意力（GQA），减少计算量
  - 旋转位置编码（RoPE），注入位置信息

  

### 2. 损失函数：交叉熵损失（Cross-Entropy Loss）

预训练的损失函数是**标准交叉熵损失**，对所有非填充 token 计算：

\mathcal{L} = \text{CrossEntropy}(\text{logits}, Y) \times \text{loss_mask}

- **logits**：模型输出的预测结果，形状 `[batch, seq_len, vocab_size]`
- **Y**：真实标签（右移后的 token ID），形状 `[batch, seq_len]`
- **loss_mask**：填充部分为 0，真实文本部分为 1，忽略填充部分的损失

**与 SFT 的关键区别**：预训练的 `loss_mask` 仅忽略填充部分，对所有真实文本 token 计算损失；而 SFT 的 `loss_mask` 还会忽略 `user`/`system` 的输入部分，仅对 `assistant` 回复计算损失。

## 五、预训练的核心训练流程

### 1. 环境与组件初始化

1. **硬件检查**：确认 GPU 显存≥16GB（推荐 24GB+），多卡训练优先。

2. 依赖加载

   ：

   - 加载 `ModelConfig`（你的 Tiny-K 配置：`dim=768`, `n_layers=12`, `vocab_size=6144`）
   - 加载预训练好的 Tokenizer
   - 加载 `PretrainDataset` 和 `DataLoader`

   

3. 模型初始化

   ：

   - 初始化 LLaMA2 模型，权重随机初始化（或加载预训练 checkpoint 继续训练）

   

4. 训练工具初始化

   ：

   - **优化器**：AdamW（学习率 `lr=5e-5` ~ `1e-4`，权重衰减 `weight_decay=0.01`）
   - **学习率调度器**：余弦退火（Cosine Annealing），带 Warmup（前 1000 步学习率线性上升）
   - **混合精度训练**：用 `Accelerate` 或 `torch.cuda.amp`，用 `fp16`/`bf16` 减少显存占用，加速训练
   - **梯度累积**：如果显存不够，用梯度累积（`gradient_accumulation_steps=8`），模拟大 batch size
   - **梯度裁剪**：`max_norm=1.0`，防止梯度爆炸

   

### 2. 训练循环（极简版）



```
for epoch in range(num_epochs):
    model.train()
    for step, (X, Y, loss_mask) in enumerate(train_dataloader):
        # 1. 前向传播
        outputs = model(X, labels=Y)
        loss = outputs.loss
        
        # 2. 应用损失掩码，忽略填充部分
        loss = (loss * loss_mask).sum() / loss_mask.sum()
        
        # 3. 反向传播（带梯度累积）
        loss = loss / gradient_accumulation_steps
        accelerator.backward(loss)
        
        # 4. 梯度裁剪 + 更新优化器
        if (step + 1) % gradient_accumulation_steps == 0:
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            optimizer.step()
            lr_scheduler.step()
            optimizer.zero_grad()
        
        # 5. 打印日志
        if step % log_steps == 0:
            print(f"Epoch {epoch}, Step {step}, Loss: {loss.item():.4f}")
    
    # 6. 验证 + 保存 checkpoint
    model.eval()
    val_loss = validate(model, val_dataloader)
    print(f"Epoch {epoch}, Val Loss: {val_loss:.4f}")
    accelerator.save_model(model, f"./checkpoint-epoch-{epoch}")
```

## 六、预训练的核心评估指标：困惑度（Perplexity）

预训练的评估指标不是 “对话效果”，而是**困惑度（Perplexity, PPL）**，这是衡量语言模型预测能力的核心指标。

### 1. 数学定义

困惑度是交叉熵损失的指数：

PPL=eL

其中 L 是验证集上的平均交叉熵损失。

### 2. 直观含义

- 困惑度越低，模型的预测越准确。

- 可以理解为 “模型预测下一个 token 时的平均候选数”：

  - PPL=10：模型平均从 10 个候选中选对下一个 token，预测很准。
  - PPL=100：模型平均从 100 个候选中选，预测较差。

  

### 3. 与 SFT 评估的区别

- **预训练评估**：只看验证集上的困惑度，越低越好。
- **SFT 评估**：除了困惑度，还要看人工评估（对话质量、指令遵循能力）、自动评估（如 MT-Bench）。

------

## 七、预训练的核心避坑指南

1. **数据质量 > 一切**：预训练的效果 90% 取决于数据，不要用脏数据训练，否则模型会学到错误的知识。
2. **不要跳过 Warmup**：训练初期学习率太高会导致模型不稳定，前 1000-5000 步用线性 Warmup，学习率从 0 上升到目标值。
3. **混合精度训练是必须的**：用 `fp16`/`bf16` 可以减少 50% 的显存占用，训练速度提升 2 倍以上。
4. **梯度裁剪防止梯度爆炸**：大模型训练时梯度很容易爆炸，设置 `max_norm=1.0` 是标准做法。
5. **定期保存 Checkpoint**：预训练通常需要几天甚至几周，每 1-2 个 epoch 保存一次 checkpoint，避免训练中断前功尽弃。

### 模块 2：生成函数 `generate`（预训练后测试用，对应预训练笔记「因果语言建模」）

```
@torch.inference_mode()  # 推理模式，禁用梯度计算，节省显存
def generate(self, idx, stop_id=None, max_new_tokens=256, temperature=1.0, top_k=None):
    """
    自回归生成：给定输入，逐token生成新文本
    """
    index = idx.shape[1]
    for _ in range(max_new_tokens):
        # 1. 截断到max_seq_len，防止超出模型上下文窗口
        idx_cond = idx if idx.size(1) <= self.args.max_seq_len else idx[:, -self.args.max_seq_len:]

        # 2. 前向传播，获取最后一个token的logits
        logits = self(idx_cond).logits
        logits = logits[:, -1, :]  # 只取最后一个时间步的输出

        # 3. 采样策略：temperature控制随机性，top_k控制候选范围
        if temperature == 0.0:
            # 贪婪采样：选概率最大的token
            _, idx_next = torch.topk(logits, k=1, dim=-1)
        else:
            logits = logits / temperature  # 缩放logits，temperature越小越确定
            if top_k is not None:
                # Top-k采样：只保留概率最高的k个token
                v, _ = torch.topk(logits, min(top_k, logits.size(-1)))
                logits[logits < v[:, [-1]]] = -float('Inf')
            probs = F.softmax(logits, dim=-1)
            idx_next = torch.multinomial(probs, num_samples=1)  # 按概率采样

        # 4. 遇到停止token就结束
        if idx_next == stop_id:
            break

        # 5. 把新token拼到输入后面，继续生成
        idx = torch.cat((idx, idx_next), dim=1)

    return idx[:, index:]  # 只返回新生成的部分
```

#### 对应预训练笔记知识点

1. **因果语言建模**：自回归生成的本质就是 “给定前面的 token，预测下一个 token”，和预训练的任务完全一致。

2. **推理模式**：`@torch.inference_mode()` 禁用梯度计算，比 `model.eval()` 更高效，节省显存。

3. 采样策略

   ：

   - `temperature=0.0`：贪婪采样，输出最确定，但容易重复。
   - `temperature>0.0` + `top_k`：随机采样，增加多样性，避免重复。

```
def get_lr(it, all):
    """
    Warmup + 余弦退火学习率调度
    """
    warmup_iters = args.warmup_iters  # 预热步数
    lr_decay_iters = all  # 总衰减步数
    min_lr = args.learning_rate / 10  # 最小学习率

    # 1. Warmup阶段：学习率从0线性增长到目标值
    if it < warmup_iters:
        return args.learning_rate * it / warmup_iters

    # 2. 超出总步数：保持最小学习率
    if it > lr_decay_iters:
        return min_lr

    # 3. 余弦退火阶段：学习率按余弦函数从目标值衰减到最小学习率
    decay_ratio = (it - warmup_iters) / (lr_decay_iters - warmup_iters)
    coeff = 0.5 * (1.0 + math.cos(math.pi * decay_ratio))  # 余弦系数，范围[0,1]
    return min_lr + coeff * (args.learning_rate - min_lr)
```

#### 对应预训练笔记知识点

1. **Warmup**：训练初期学习率太高会导致模型不稳定，前 N 步线性增长，对应预训练笔记里的「不要跳过 Warmup」。
2. **余弦退火**：学习率按余弦函数平滑衰减，比线性衰减更稳定，能帮助模型更好地收敛到局部最优。
3. **最小学习率**：防止学习率衰减到 0，保持模型的微调能力。

        X = X.to(args.device)
        Y = Y.to(args.device)
        loss_mask = loss_mask.to(args.device)

        # 2. 动态调整学习率
        lr = get_lr(epoch * iter_per_epoch + step, args.epochs * iter_per_epoch)
        for param_group in optimizer.param_groups:
            param_group['lr'] = lr

        # 3. 混合精度训练上下文
        with ctx:
            # 前向传播
            out = model(X, Y)
            loss = out.last_loss / args.accumulation_steps  # 除以累积步数
            # 应用损失掩码：忽略padding部分
            loss_mask = loss_mask.view(-1)
            loss = torch.sum(loss * loss_mask) / loss_mask.sum()

        # 4. 混合精度反向传播
        scaler.scale(loss).backward()

        # 5. 梯度累积：每accumulation_steps步更新一次优化器
        if (step + 1) % args.accumulation_steps == 0:

            # 梯度裁剪：防止梯度爆炸
            torch.nn.utils.clip_grad_norm_(model.parameters(), args.grad_clip)
            # 更新优化器
            scaler.step(optimizer)
            # 更新scaler的缩放因子
            scaler.update()
            # 清零梯度
            optimizer.zero_grad(set_to_none=True)

        # 6. 日志记录
        if step % args.log_interval == 0:

        # 7. 模型保存
        if (step + 1) % args.save_interval == 0:

#### 对应预训练笔记知识点

1. **损失掩码**：`loss = torch.sum(loss * loss_mask) / loss_mask.sum()`，只对真实文本部分计算损失，忽略 padding，对应预训练笔记里的「损失掩码」。
2. **混合精度训练**：`torch.cuda.amp.autocast()` + `GradScaler`，用`fp16`/`bf16`减少显存占用，加速训练，对应预训练笔记里的「混合精度训练是必须的」。
3. **梯度累积**：`loss / args.accumulation_steps` + 每 N 步更新一次优化器，模拟大 batch size，解决显存不够的问题，对应预训练笔记里的「梯度累积」。
4. **梯度裁剪**：`torch.nn.utils.clip_grad_norm_(model.parameters(), args.grad_clip)`，防止梯度爆炸，对应预训练笔记里的「梯度裁剪防止梯度爆炸」。
