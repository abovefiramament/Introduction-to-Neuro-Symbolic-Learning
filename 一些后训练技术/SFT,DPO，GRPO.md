SFT、DPO、GRPO 核心详解：大模型对齐技术的完整演进
先给你一个贴合考核与落地需求的总览定位：这三个技术是大语言模型从「通用预训练基座」到「符合人类需求 / 指令 / 推理偏好的可用模型」的核心对齐技术，呈现清晰的递进演进关系：

SFT：对齐的基础，把预训练模型的「文本续写能力」转化为「指令遵循能力」，是所有后续对齐的前提；
DPO：主流的偏好对齐方案，用极简的方式替代了传统 RLHF 的复杂 PPO 流程，解决 SFT 无法对齐人类细粒度偏好的痛点；
GRPO：专为复杂推理任务优化的进阶对齐技术，是 DPO 的升级方案，完美适配你之前学习的 CoT 思维链推理场景，也是当前 OpenAI o1 系列、开源推理模型的核心对齐方案。

三者在大模型完整训练流程中的位置：预训练基座模型 → SFT 有监督微调 → 偏好对齐（DPO/GRPO）→ 推理部署
下面结合你之前学习的 KL 散度、CoT、自我一致性等知识体系，从核心定义、数学原理、执行流程、优缺点、落地适配五个维度，做完整拆解。

一、SFT：Supervised Fine-Tuning 有监督微调
1. 核心定义与定位
SFT 是用高质量的「指令 - 答案」配对数据，对预训练基座模型做全参数 / 轻量化微调，让模型学会遵循人类指令、输出符合要求格式的内容，是预训练之后对齐的第一步，也是所有后续对齐技术的基础。
预训练模型的核心能力是「文本续写」，只会根据上文补全下文，无法理解 “问答、总结、推理、创作” 等人类指令；而 SFT 的核心目标，就是把模型的通用语言能力，转化为可交互的指令遵循能力。
2. 核心原理与数学损失
SFT 的本质是有监督的自回归语言建模，核心损失是标准的交叉熵损失（Cross-Entropy Loss）。
对于一条标注数据：指令Prompt → 标准答案Response，模型的优化目标是：最大化标准答案序列的生成概率。数学公式：
LSFT=−N1∑i=1Nlogpθ(yi∣y<i,x)其中：

x：输入的指令 Prompt；
yi：标准答案 Response 的第i个 token；
pθ：微调后模型的生成概率；
N：标准答案的 token 总长度。

简单来说：模型每一步生成 token 时，都要让标准答案里的对应 token 概率最大化，强制模型学习标注数据里的指令响应模式。
3. 完整执行流程

数据准备：构建高质量的指令微调数据集，核心是「指令 + 标准答案」的配对数据，包括通用对话、知识问答、代码生成、CoT 推理等场景；需要做严格的数据清洗、去重、格式标准化，避免噪声数据导致模型学偏。
训练配置：主流采用LoRA/QLoRA 轻量化微调（冻结基座模型，只微调少量适配器参数），降低显存占用与训练成本；极少数场景用全参数微调。
模型微调：用上述交叉熵损失，在指令数据集上训练模型，让模型拟合标注的指令 - 答案映射关系。
效果验证：通过人工评估、自动指标（如 Rouge、BLEU）、下游任务测试，验证模型的指令遵循能力、回答质量、推理能力。

4. 核心优缺点
表格优势局限性流程简单、训练稳定、落地门槛极低，是开源模型微调的标配高度依赖标注数据的质量，数据差则模型效果差能快速让模型具备基础的指令遵循能力，适配绝大多数基础交互场景只能学习标注的 “标准答案”，无法对齐人类的细粒度偏好（比如 “更简洁的回答”“更严谨的 CoT 推理”）可通过 CoT 标注数据，快速提升模型的基础推理能力容易过拟合，标注数据覆盖不到的场景，模型表现会大幅下降；存在 “对齐税”（指令遵循能力提升，通用能力下降）
5. 与你之前知识的关联
你之前学习的CoT 思维链、生成知识提示 GKP，核心落地环节就是 SFT：

在 SFT 阶段，用大量「问题 + 完整 CoT 推理过程 + 答案」的标注数据微调模型，能让模型自发学会用 CoT 做复杂推理，解锁推理涌现能力；
用 GKP 的两阶段数据做 SFT 微调，能让模型养成「先沉淀知识、再做推理」的习惯，大幅减少推理幻觉。


二、DPO：Direct Preference Optimization 直接偏好优化
1. 核心定义与定位
DPO 是 2023 年提出的基于人类偏好的端到端对齐技术，核心是替代传统 RLHF（基于人类反馈的强化学习）中复杂、不稳定、高成本的 PPO（近端策略优化）流程，用一个极简的分类损失，直接实现模型与人类偏好的对齐。
它的定位是SFT 之后的进阶对齐：SFT 只能让模型学会 “怎么回答”，而 DPO 能让模型学会 “怎么回答得更好”—— 比如更严谨的 CoT、更少的幻觉、更符合人类偏好的语气 / 格式 / 逻辑。
2. 核心原理与数学推导（关联你学过的 KL 散度）
DPO 的核心创新是：从理论上证明了 RLHF 的奖励建模 + PPO 强化学习两步流程，可以等价转化为一个直接的、基于成对偏好数据的分类损失，完全不需要单独训练奖励模型、也不需要复杂的强化学习流程。
核心前置逻辑
RLHF 的核心优化目标是：在保证模型与 SFT 参考模型的分布偏差不太大（避免模式崩溃）的前提下，最大化人类对模型输出的奖励。数学上的优化目标为：
maxθE(x,y)∼pθ[r(x,y)]−β⋅DKL(pθ(y∣x)∥pref(y∣x))其中：

r(x,y)：人类对模型输出的奖励；
β：KL 散度约束系数，避免模型偏离 SFT 参考模型太远，防止模式崩溃；
DKL：你之前学习的正向 KL 散度，以 SFT 参考模型pref为基准，约束微调模型pθ的分布。

DPO 的核心突破，是通过奖励模型与策略模型的数学等价变换，把这个强化学习优化目标，直接转化为了一个仅需要成对偏好数据的简单损失函数：
LDPO=−E(x,yw,yl)∼D[logσ(β⋅logpref(yw∣x)pθ(yw∣x)−β⋅logpref(yl∣x)pθ(yl∣x))]其中：

yw：人类偏好的 “优胜答案”（比如更严谨的 CoT、更准确的回答）；
yl：人类不偏好的 “劣败答案”（比如逻辑错误的 CoT、有幻觉的回答）；
σ：Sigmoid 激活函数；
pref：SFT 训练后的参考模型（固定权重，不参与训练）。

简单来说：DPO 会强制模型让「优胜答案」的相对概率越来越高，「劣败答案」的相对概率越来越低，同时通过 KL 散度约束保证模型不偏离 SFT 的基础能力，完美实现偏好对齐。
3. 完整执行流程

前置准备：完成 SFT 微调，得到 SFT 模型，作为 DPO 的固定参考模型pref。
偏好数据准备：构建成对偏好数据集—— 对同一个指令，准备「优胜答案yw+ 劣败答案yl」的配对数据，数据来源可以是人工标注、模型自动排序、自我一致性 SC 筛选。
模型微调：用上述 DPO 损失函数，对 SFT 模型做轻量化微调，固定参考模型权重，仅更新待优化模型的参数。
效果验证：通过人工偏好评估、下游推理任务测试，验证模型是否对齐了目标偏好（比如 CoT 更严谨、幻觉更少）。

4. 与传统 PPO 的核心区别
表格维度DPOPPO（传统 RLHF）训练流程单阶段端到端训练，仅需成对偏好数据三阶段流程：SFT→奖励模型训练→PPO 强化学习计算成本极低，和 SFT 基本持平极高，需要 4 倍以上的算力，训练不稳定稳定性极高，无强化学习的梯度爆炸、模式崩溃问题极低，超参数敏感，极易出现训练崩溃、奖励黑客落地门槛极低，开源工具链成熟极高，需要复杂的工程化适配
5. 核心优缺点
表格优势局限性流程极简、训练稳定、计算成本低，是当前开源模型偏好对齐的绝对主流方案高度依赖成对偏好数据，数据标注成本依然较高完美解决 PPO 的不稳定性问题，无模式崩溃、奖励黑客风险仅能做整条序列的整体偏好对齐，无法针对 CoT 的单步推理做细粒度优化可精准对齐人类的细粒度偏好（推理严谨性、回答简洁性、幻觉规避等）存在轻微的对齐税，偏好对齐后，模型的通用创作能力可能小幅下降

三、GRPO：Group Relative Preference Optimization 分组相对偏好优化
1. 核心定义与定位
GRPO 是 OpenAI 在 o1 推理模型中提出、并由 DeepSeek 开源验证的专为复杂 CoT 推理任务优化的进阶偏好对齐技术，是 DPO 的升级方案，核心解决了 DPO 在推理任务上的核心痛点：无法做细粒度的推理步骤优化、依赖成对偏好数据、对复杂多步推理的优化效果有限。
它的核心定位是CoT 推理能力的专项对齐：DPO 只能让模型学会 “哪个答案更好”，而 GRPO 能让模型学会 “哪一步推理更正确”，从根源上优化模型的多步推理能力，是当前开源推理模型（如 DeepSeek-R1）的核心对齐方案。
2. 核心原理与四大创新点
GRPO 的核心逻辑和你之前学习的自我一致性 SC高度同源：对同一个问题，采样多条独立的 CoT 推理路径，通过组内路径的相对奖励，做细粒度的策略更新，不需要成对标注数据、不需要单独的参考模型，就能实现推理能力的精准优化。
它的四大核心创新，完美解决了 DPO 的痛点：

单问题分组采样，无需成对偏好数据：对同一个问题，采样 N 条独立的 CoT 推理路径，形成一个路径组，不需要人工标注 “优胜 / 劣败” 配对数据，大幅降低标注成本。
相对奖励归一化，降低训练方差：用组内所有路径的奖励均值做归一化，得到相对奖励，而非绝对奖励，大幅降低了奖励计算的方差，训练稳定性远超 PPO/DPO。
逐 token 细粒度权重更新：针对 CoT 的每一个 token，计算其对最终结果的贡献权重，只优化对正确推理有帮助的 token，而非整条序列，精准修正推理步骤的错误。
内置 KL 散度约束，无需参考模型：在损失函数中内置了 KL 散度约束，用模型自身的初始概率作为参考，不需要单独的 SFT 参考模型，进一步简化了训练流程。

3. 核心数学逻辑
GRPO 的优化目标是最大化相对奖励加权的对数概率，同时约束 KL 散度避免模式崩溃，核心损失函数为：
LGRPO=−Ex∼D[N1∑i=1N∑t=1T(r^i−β⋅logpref(yi,t∣x,yi,<t)pθ(yi,t∣x,yi,<t))⋅logpθ(yi,t∣x,yi,<t)]其中核心参数：

N：单个问题采样的 CoT 路径数量（通常为 16/32/64 条）；
r^i：第i条路径的相对奖励，计算公式为 r^i=ri−N1∑j=1Nrj（ri为该路径的绝对奖励，如答案是否正确、CoT 逻辑是否严谨）；
β：KL 散度约束系数；
pref：模型更新前的初始概率，无需单独的参考模型。

4. 完整执行流程

前置准备：完成 SFT 微调，得到具备基础 CoT 推理能力的模型。
分组采样：对同一个问题 Prompt，通过温度采样生成N条独立的 CoT 推理路径 + 最终答案，形成一个路径组。
绝对奖励打分：对每条路径，通过自动评分函数（如答案是否正确、代码是否可运行、CoT 步骤是否完整）给出 0~1 的绝对奖励ri，无需人工标注。
相对奖励计算：用组内奖励均值做归一化，得到每条路径的相对奖励r^i，消除奖励的整体偏移。
逐 token 梯度更新：对每条路径的每一个 token，用相对奖励加权，计算策略梯度，更新模型参数，同时通过 KL 散度约束避免模式崩溃。
迭代训练：重复上述采样 - 打分 - 更新流程，直到模型的推理能力达到目标。

5. 核心优缺点
表格优势局限性专为 CoT 推理优化，能精准修正单步推理错误，大幅提升复杂数学、代码、逻辑推理能力核心适配推理任务，开放式对话、创意生成等场景的对齐效果不如 DPO无需成对偏好标注数据，仅需自动奖励函数，标注成本几乎为零对奖励函数的设计要求极高，奖励函数的质量直接决定最终效果训练稳定、计算成本低，仅需 DPO 的 1.5 倍算力，效果远超 DPO/PPO单轮训练需要采样多条路径，推理成本高于 DPO对齐税极低，优化推理能力的同时，完全不损失模型的通用语言能力开源工具链成熟度略低于 DPO，落地门槛稍高
6. 与你之前知识的关联
GRPO 和你之前学习的自我一致性 SC是天然互补的：

自我一致性 SC 的核心是「多路径采样 + 多数投票」，筛选出最可靠的答案；
GRPO 的核心是「多路径采样 + 相对奖励 + 细粒度更新」，用多路径的差异直接优化模型的推理能力；
两者结合可以形成闭环：用 SC 筛选高质量的 CoT 路径，再用 GRPO 做模型的推理对齐，是当前推理模型训练的 SOTA 方案。

五、常见误区澄清（考核高频易错点）

误区 1：DPO/GRPO 可以替代 SFT纠正：绝对不能。SFT 是所有对齐的基础，DPO/GRPO 必须基于 SFT 模型做优化 —— 预训练基座模型没有指令遵循能力，直接做 DPO/GRPO 会完全失效，甚至破坏模型的基础语言能力。

误区 2：GRPO 是 DPO 的替代方案，全面超越 DPO纠正：GRPO 是 DPO 在推理场景的专项升级，而非全面替代。通用对话、合规性对齐、创意生成等场景，DPO 的效果依然优于 GRPO，且落地更简单。

误区 3：SFT 数据越多，模型效果越好纠正：SFT 的核心是数据质量，而非数量。1 万条高质量、多样化的指令数据，效果远好于 100 万条低质量、重复的噪声数据，过量低质量数据会导致模型过拟合、通用能力下降。
