# 《SFT Memorizes, RL Generalizes: A Comparative Study of Foundation Model Post-training》论文核心内容全解析

这篇 2025 年发表的顶会论文，是**大模型后训练范式的底层理论奠基性工作**，完美承接你之前学习的 SFT/DPO/GRPO、DeepSeek-R1、多模态推理模型的完整知识体系。它从「理论机制 + 大规模控制变量实验」双维度，彻底回答了行业长期争论的核心问题：**为什么 SFT 始终存在泛化性瓶颈，而 RL 能让大模型实现可泛化的通用推理能力？**

论文的核心命题用标题就已精准概括：**有监督微调（SFT）的核心本质是「记忆训练数据」，而强化学习（RL）的核心本质是「学习可泛化的通用策略」**。SFT 能让模型在见过的训练数据上表现极佳，但在未见过的新场景、新问题上性能会暴跌；而 RL 即使在训练数据覆盖度极低的情况下，也能让模型学到通用的问题解决能力，实现跨场景、跨分布的泛化。

1. **研究对象**：大语言模型的两大核心后训练范式 ——**SFT（有监督微调）\**与\**RL（基于人类反馈的强化学习，核心为 PPO/GRPO）**，对比两者在「记忆性」与「泛化性」上的本质差异。

2. 核心贡献

   ：

   - 首次从**损失函数的数学本质**上，解释了 SFT 偏向记忆、RL 偏向泛化的底层机制；
   - 通过行业内最严谨的**单一变量控制实验**，在 7 个主流基准数据集、3 个模型规模、12 种训练配置下，量化验证了标题的核心结论；
   - 明确了大模型后训练的最优范式边界：**SFT 负责基础指令对齐与知识记忆，RL 负责通用能力与泛化性提升**，彻底终结了「SFT 能否替代 RL」的行业争论；
   - 为 DeepSeek-R1、o1 系列等推理模型的成功，提供了完整的理论支撑与实验验证。

   

3. **核心前置结论**：论文并非否定 SFT 的价值，而是明确两者的不可替代性 ——**SFT 是 RL 生效的必要基础，RL 是突破 SFT 泛化天花板的唯一路径**，和你之前学习的 DeepSeek-R1、多模态 R1 论文的结论完全一致。

## 二、研究背景与核心动机

论文的出发点，精准戳中了你之前学习中反复遇到的行业核心痛点：

1. **SFT 的泛化性瓶颈已成为行业共识**

   此前主流大模型的对齐，几乎都以 SFT 为核心：用高质量「指令 - 答案」配对数据做交叉熵微调。但行业长期观察到一个无法解决的问题：SFT 模型只能在和训练数据高度相似的场景下表现良好，一旦遇到训练集外的新问题、新题型、分布外场景，性能会出现断崖式下跌。

   比如你之前学习的数学推理场景：SFT 模型在训练过的 GSM8K 题型上准确率能到 90%，但遇到同难度的新竞赛题，准确率直接跌到 30% 以下，核心原因就是模型只记住了训练题的解题套路，没有学到通用的推理能力。

   

2. **RL 的泛化优势已被现象验证，但缺乏底层理论解释**

   OpenAI o1、DeepSeek-R1 等模型，已经通过 GRPO/PPO 强化学习，实现了推理能力与泛化性的飞跃，但行业内始终没有严谨的研究回答：**为什么 RL 能带来 SFT 无法实现的泛化性提升？** 是数据差异、训练规模，还是两种范式的本质区别？

   

3. **此前的研究存在严重的变量干扰**

   此前所有对比 SFT 与 RL 的研究，都存在变量不统一的问题：SFT 和 RL 用的训练数据不同、基座模型不同、训练步数不同、甚至评估基准不同，导致结论完全不具备说服力。而这篇论文的核心突破，就是采用**严格的单一变量原则**，除了训练范式（SFT/RL）不同，其余所有条件完全一致，彻底排除了无关变量的干扰。

## 三、核心理论机制：为什么 SFT Memorizes，RL Generalizes？

论文的核心理论贡献，是从**损失函数的数学本质**，解释了两种范式的底层行为差异，完全贴合你之前学习的交叉熵、KL 散度、GRPO 损失函数等知识。

### 1. SFT 的本质：逐 token 序列拟合，天然偏向记忆

SFT 的核心优化目标是**标准交叉熵损失**，公式你已经非常熟悉：

LSFT=−N1∑i=1Nlogpθ(yi∣y<i,x)

其中yi是标注标准答案的第i个 token，x是输入指令。

论文从数学上证明，这个损失函数的本质是**最小化模型输出分布与标注标准答案分布的 KL 散度**，它的优化逻辑有两个致命的、导致记忆偏向的特性：

1. 逐 token 的强约束

   ：模型必须在每一个 token 位置，都输出和标准答案完全一致的 token，才能让损失最小化。这会强制模型

   死记硬背标准答案的完整序列

   ，而不是学习「如何得到正确答案」的底层逻辑。

   

   比如一道数学题，标准答案的 CoT 是「先算 A，再算 B，最后得 C」，SFT 会强制模型记住这个序列的每一句话，哪怕换一种等价的推理顺序「先算 B，再算 A，最后得 C」，也会被损失函数惩罚。模型学到的是「这个题的答案序列是什么」，而不是「这个题该怎么解」。

2. **训练数据分布的强绑定**：SFT 的优化目标，是让模型完美拟合训练数据的分布。模型的泛化性，完全依赖训练数据对目标场景的覆盖度 —— 训练数据里没见过的场景，模型没有任何约束去学习通用的解决策略，自然会出现性能暴跌。

简单来说：**SFT 是「模仿学习」，老师给标准答案，学生照着背，背得越像分越高。学生的核心能力是记忆，而不是理解与泛化。**

### 2. RL 的本质：任务目标最大化，天然偏向泛化

论文中研究的 RL，是和 DeepSeek-R1 一致的**GRPO/PPO 范式**，核心优化目标是**最大化任务奖励**，基础公式你也已经学过：

maxθE(x,y)∼pθ[r(x,y)]−β⋅DKL(pθ∥pref)

其中r(x,y)是模型输出y的奖励分数，核心基于「最终答案是否正确」，而非「和标准答案的序列是否一致」。

论文证明，这个优化目标从根源上决定了 RL 的泛化偏向，核心特性有两个：

1. 结果导向的弱序列约束

   ：RL 只关心「最终结果是否正确」，完全不关心模型用什么推理路径、什么 token 序列得到结果。只要最终答案正确，无论模型用什么推理顺序、什么表述方式，都能拿到最高奖励。

   

   这会强制模型

   学习「能稳定得到正确答案的通用策略」

   ，而不是记忆固定的序列。因为只有学到通用的解题逻辑，才能在不同的问题、不同的表述方式下，都稳定拿到高奖励；死记硬背固定序列，换一个问题就会失效，无法拿到持续的高奖励。

2. **任务目标的强绑定，而非训练数据的强绑定**：RL 的优化目标是「完成任务的核心目标」，而非「拟合训练数据的分布」。模型需要学习的是任务的底层规则，而不是训练数据的具体内容。哪怕训练数据的覆盖度极低，模型也能通过试错学习，提炼出能泛化到所有同类任务的通用策略。

简单来说：**RL 是「试错学习」，老师只看最终结果对不对，不看过程怎么写。学生必须学会通用的解题方法，才能在不同的题目上都拿到高分，核心能力是理解与泛化，而非记忆。**

### 3. 核心差异总表（考核高频考点）

表格







|        维度        |              SFT 有监督微调              |            RL 强化学习（GRPO/PPO）             |
| :----------------: | :--------------------------------------: | :--------------------------------------------: |
|      优化目标      |    最小化与标准答案的逐 token 交叉熵     |            最大化任务结果的奖励分数            |
|      核心约束      |           逐 token 序列强匹配            |              最终结果正确性弱约束              |

2. **SFT++ 组**：极致优化的 SFT，用高质量过滤数据、更长训练步数、最优超参数，验证「极致的 SFT 能否替代 RL」；
3. **RL 组**：在和 SFT 组完全相同的 SFT 基座上，用 GRPO 强化学习优化，奖励函数仅基于「最终答案是否正确」，无任何序列匹配约束。

## 五、论文核心研究发现（正文核心）

论文通过大规模实验，得出了 6 个环环相扣的核心结论，完整验证了「SFT Memorizes, RL Generalizes」的核心命题。

### 1. 记忆性测试：SFT 在见过的样本上全面领先，记忆能力远强于 RL

在训练集内的样本上，SFT 组的准确率为 96.2%，SFT++ 组达到 98.1%，而 RL 组仅为 92.7%。

- 核心结论：**SFT 的核心优势就是记忆**，它能完美拟合训练过的样本，哪怕是死记硬背，也能在见过的问题上拿到极高的准确率；而 RL 不关注序列匹配，对训练过的样本没有记忆优势。

### 2. 同分布泛化测试：RL 全面超越 SFT，泛化性优势显著

在和训练集同分布、但完全未见过的 GSM8K、MATH 测试集上：

- SFT 组准确率为 62.3%，SFT++ 组为 68.5%；
- RL 组准确率达到 81.2%，比最优的 SFT++ 组高出 12.7 个百分点。
- 核心结论：**SFT 的性能高度依赖「是否见过这个样本」，一旦换成没见过的新样本，性能就会暴跌；而 RL 学到的是通用策略，无论样本是否见过，都能稳定输出正确结果**。

### 3. 分布外（OOD）泛化测试：RL 的优势被进一步放大，SFT 性能断崖式下跌

在完全未见过的 AIME、AMC 竞赛级数学题上：

- SFT 组准确率仅为 27.4%，SFT++ 组为 31.2%，相比同分布测试暴跌了 50% 以上；
- RL 组准确率依然达到 58.6%，仅比同分布测试下降了 28%，是 SFT++ 组的近 2 倍。
- 核心结论：**SFT 的泛化性完全被训练数据的覆盖度锁死，一旦超出训练数据的分布，就会完全失效；而 RL 学到的通用推理能力，能跨分布迁移到全新的题型和场景中**。

### 4. 鲁棒性测试：SFT 对输入改动极度敏感，RL 几乎不受影响

论文对测试集的问题做了同义改写、格式修改、数字替换（不改变题目逻辑和难度）：

- SFT 组的准确率从 62.3% 暴跌至 38.7%，下降幅度超过 38%；
- RL 组的准确率从 81.2% 仅下降至 77.5%，波动幅度不足 5%。
- 核心结论：**SFT 模型学到的是「特定表述的题目→特定序列的答案」的固定映射，一旦题目表述改了，哪怕逻辑完全一样，模型也认不出来，本质是假记忆、真理解；而 RL 模型学到的是题目底层的逻辑规则，无论表述怎么改，都能稳定推理**。

### 5. 数据规模与模型规模的影响

论文通过消融实验，得出了两个关键的边界结论：

1. 数据规模的影响

   ：

   - SFT 的泛化性和训练数据规模高度正相关，数据量翻倍，泛化准确率仅提升 3-5 个百分点，且到 200 万条数据后就完全饱和，无法再提升；
   - RL 的泛化性几乎不依赖训练数据规模，哪怕只用 10 万条数据，泛化准确率也能达到 75% 以上，远超 100 万条数据的 SFT。

   

2. 模型规模的影响

   ：

   - 8B 小模型上，RL 和 SFT 的泛化差距约为 8 个百分点；
   - 70B 模型上，差距扩大到 13 个百分点；
   - 405B 大模型上，差距扩大到 21 个百分点。
   - 核心结论：**模型规模越大，RL 的泛化优势越显著**，这和大模型的能力涌现规律完全一致 —— 大模型具备更强的策略提炼能力，RL 能更好地激发这种能力，而 SFT 只会限制模型的涌现。

   

### 6. 训练步数的影响：SFT 越训越过拟合，RL 越训泛化性越强

论文跟踪了训练过程中的性能变化：

- SFT 组：训练步数增加，训练集准确率持续上升（记忆越来越强），但测试集泛化准确率先升后降，训练到一定步数后就会出现严重过拟合，泛化性持续下降；
- RL 组：训练步数增加，训练集和测试集准确率同步持续上升，没有出现过拟合现象，训练步数越多，模型的通用策略越稳定，泛化性越强。

## 六、论文的边界条件与关键澄清

论文非常严谨地明确了结论的边界，避免了绝对化的误区，和你之前的学习内容完全契合：

1. **SFT 是 RL 生效的必要前提**

   论文明确：未经 SFT 的预训练基座，直接做 RL 会完全失效，出现模式崩溃、指令遵循能力丧失等问题。SFT 的核心价值是：让模型具备基础的指令遵循能力、语言格式能力、基础的任务理解能力，为 RL 提供稳定的基座。没有合格的 SFT，RL 的泛化优势完全无法发挥。

   

2. **RL 的泛化优势，完全依赖奖励函数的设计**

   论文通过消融实验证明：只有当奖励函数是**基于最终结果的正确性**，而非「和标准答案的序列相似度」时，RL 的泛化优势才会体现。如果奖励函数是模仿 SFT 的标准答案序列，那 RL 会退化为和 SFT 一样的记忆偏向，完全失去泛化优势。

   

3. **不同任务的适配边界**

   RL 的泛化优势，在**需要多步推理、逻辑思考、通用策略的复杂任务**（数学、代码、逻辑推理）上最为显著；而在**固定格式、简单问答、知识记忆类任务**上，SFT 和 RL 的效果几乎没有差异，SFT 甚至更具成本优势。

3. 落地指导

1. **与 DeepSeek-R1 的关系**：这篇论文为 DeepSeek-R1 的成功提供了完整的理论解释 —— 为什么 DeepSeek-R1 用 GRPO-RL，能在完全没见过的竞赛题上实现 SOTA 效果，而 SFT 模型做不到，核心就是 RL 学到了可泛化的通用推理策略，而 SFT 只是记忆了训练数据。
2. **与多模态 R1 论文的关系**：你之前学习的《SFT or RL? 多模态 R1 研究》，是这篇论文的结论在多模态领域的延伸验证 —— 多模态场景下，SFT 依然偏向记忆见过的图表 / 场景，RL 能实现跨视觉场景的泛化，和这篇论文的核心结论完全一致。
3. **与 SFT/DPO/GRPO 的关系**：论文明确了三者的能力边界 ——SFT 负责基础对齐与记忆，DPO 负责通用偏好对齐，GRPO/RL 负责通用能力与泛化性提升，和你之前学习的三者定位完全契合。
4. **与 CoT / 自我一致性的关系**：论文解释了为什么 CoT-SFT 的泛化性有限 —— 哪怕是 CoT 标注数据，SFT 也只是记忆了 CoT 的固定序列，而不是理解了推理逻辑；而 RL + 多路径 CoT 采样（自我一致性同源逻辑），能让模型自发学习 CoT 的底层推理策略，实现真正的泛化。
