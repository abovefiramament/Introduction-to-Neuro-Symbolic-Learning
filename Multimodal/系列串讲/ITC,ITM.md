# 一、使用交叉注意力机制的经典 / 前沿多模态模型（除 Qwen-VL 外）

首先先明确核心定义：多模态领域的**交叉注意力（Cross-Attention）**，核心特征是**Query（Q）来自一个模态，Key（K）和 Value（V）来自另一个模态**，实现跨模态的特征深度交互与语义对齐，和你之前学习的 Transformer Encoder-Decoder 交叉注意力逻辑完全一致。

下面按「经典预训练基座→多模态对话大模型→跨模态生成模型」的体系分类，覆盖你考核的核心范围，同时标注每个模型交叉注意力的核心用法：

## （一）经典视觉 - 语言预训练基座（交叉注意力的核心落地源头）

这类模型是多模态交叉注意力的奠基者，也是 Qwen-VL 等后续模型的技术基础，考核中高频出现。

1. ALBEF

   

   多模态预训练的里程碑模型，首次系统提出「ITC+ITM+MLM」三损失预训练范式，也是你第二个问题的核心出处。

   - 交叉注意力用法：采用「单塔融合结构」，文本特征作为 Q，图像特征作为 K/V，通过交叉注意力实现图文的深度交互，是后续 ITM 任务的核心基础。

   

2. BLIP / BLIP-2

   

   ALBEF 的进阶版，是目前开源多模态模型的主流技术底座。

   - BLIP：优化了交叉注意力的双路设计，一路用交叉注意力做图文理解（VQA、图文匹配），一路用编码器 - 解码器交叉注意力做图文生成（图像描述）；
   - BLIP-2：核心创新**Q-Former**，和你上一轮学习的 Qwen-VL 视觉特征压缩模块完全一致 —— 用可学习的 Query 作为 Q，图像特征作为 K/V，通过交叉注意力把 1000 + 的视觉 token 压缩为固定数量的低维 token，实现视觉特征和 LLM 的高效对齐，是目前工业界最主流的交叉注意力压缩方案。

   

3. Flamingo

   

   DeepMind 推出的「大语言模型 + 多模态」的奠基之作，首次实现了冻结 LLM + 视觉编码器的高效多模态适配。

   - 交叉注意力用法：在 LLM 的 Decoder 层中插入**门控交叉注意力层**，视觉编码器的输出作为 K/V，LLM 的文本序列特征作为 Q，把视觉信息精准注入大语言模型，几乎所有后续闭源 / 开源多模态大模型都沿用了这套设计。

   

4. FLAVA

   

   Meta 推出的通用多模态基座，用统一的 Transformer 架构覆盖图文理解、检索、生成全任务。

   - 交叉注意力用法：采用统一的跨模态 Transformer，通过交叉注意力实现图像、文本的双向交互，支持单模态、跨模态的多任务学习。

   

> 注意：**CLIP 没有使用交叉注意力**，它是双塔对比学习架构，视觉和文本编码器完全独立，仅在损失计算时做全局特征对比，没有跨模态的特征交互，这是高频考点的易错点。

## （二）开源多模态对话大模型（与 Qwen-VL 同赛道）

1. InternVL 系列

   

   国产开源多模态 SOTA 模型，核心采用

   分层交叉注意力架构

   ，通过交叉注意力实现高分辨率图像特征和 LLM 的高效对齐，解决了长文档、高分辨率图像的理解痛点。

2. DeepSeek-VL 系列

   

   采用「视觉编码器 + 交叉注意力适配器 + LLM」的架构，通过轻量的交叉注意力层，实现细粒度的图文特征交互，在 OCR、文档理解任务上表现突出。

3. MiniGPT-4 v2

   

   在初代线性投影的基础上，引入了交叉注意力层优化视觉 - 文本对齐，解决了初代模型图文对齐不精准、细节理解差的问题。

4. GPT-4V / Gemini 系列（闭源）

   

   业内公认采用了多层交叉注意力架构，实现图像、文本、音频、视频等多模态的深度交互，是目前多模态大模型的技术标杆。

## （三）其他跨模态场景的经典交叉注意力应用

1. Stable Diffusion 系列（文生图）

   

   交叉注意力在生成领域的最经典应用，也是目前 AIGC 的核心技术之一。

   - 用法：UNet 的扩散模型中，文本编码器（CLIP）输出的 prompt 特征作为 K/V，图像潜空间特征作为 Q，通过交叉注意力实现文本对图像生成的精准控制，让模型生成符合 prompt 描述的内容。

   

2. Whisper（语音 - 文本识别）

   

   OpenAI 的语音识别模型，采用标准的 Transformer Encoder-Decoder 架构。

   - 用法：语音 Encoder 的输出作为 K/V，文本 Decoder 的序列特征作为 Q，通过交叉注意力实现语音特征和文本的对齐，完成语音到文本的转写。

   

3. VideoLLaMA / VideoBLIP（视频 - 文本理解）

   

   视频多模态模型，通过交叉注意力实现视频帧特征和文本的时序 - 空间双向对齐，解决长视频的理解、问答需求。

# 二、ITC 与 ITM 核心原理详解

ITC 和 ITM 是**视觉 - 语言预训练的两大核心损失函数**，最早在 ALBEF 中被系统提出并形成标准范式，后续 BLIP、Qwen-VL 等几乎所有多模态预训练模型均沿用，核心目标是实现视觉与文本的双向语义对齐，两者通常搭配使用，分别负责「全局粗对齐」和「细粒度精对齐」。

## （一）ITC：Image-Text Contrastive Loss 图文对比损失

### 1. 核心目标

让**匹配的图文对（正样本）在特征空间中距离尽可能近，不匹配的图文对（负样本）距离尽可能远**，实现跨模态的全局语义级对齐，核心继承自 CLIP 的对比学习思想。

### 2. 定量计算逻辑

以一个批次（batch）包含N个独立的图文对为例：

1. 对每个图文对，分别用视觉编码器得到图像全局特征vi∈RD，用文本编码器得到文本全局特征ti∈RD（D为特征维度，保证两个模态特征维度一致）；
2. 计算批次内所有图像和文本特征的两两余弦相似度，得到N×N的相似度矩阵S，其中\mathbf{S}_{i,j} = \text{cos_sim}(\mathbf{v}_i, \mathbf{t}_j)；
3. 相似度矩阵中，**对角线元素\**Si,i\**为正样本**（匹配的图文对），**非对角线元素\**Si,j(i=j)\**为负样本**（不匹配的图文对）；
4. 分别计算「图像到文本（I2T）」和「文本到图像（T2I）」的双向 InfoNCE 损失，最终取平均值作为 ITC 损失：LI2T=−N1∑i=1Nlog∑j=1Nexp(Si,j/τ)exp(Si,i/τ)LT2I=−N1∑i=1Nlog∑j=1Nexp(Sj,i/τ)exp(Si,i/τ)LITC=21(LI2T+LT2I)其中τ为温度系数，用于控制相似度的分布锐度，是超参数。

### 3. 核心作用与特点

- 核心作用：学习全局的图文语义关联，比如让 “猫” 的图像和 “一只猫” 的文本在特征空间中靠近，和 “一只狗” 的文本远离，是多模态预训练最基础的对齐损失；
- 特点：双塔结构，视觉和文本编码器独立计算，仅在损失计算时交互，计算效率极高，适合大规模图文对预训练；
- 局限性：仅做全局特征的粗对齐，无法学习细粒度的图文对应关系（比如图像中 “猫的耳朵” 和文本中 “猫的耳朵” 的对应）。

## （二）ITM：Image-Text Matching Loss 图文匹配损失

### 1. 核心目标

让模型学习判断「图文对是否匹配」，通过交叉注意力实现图文的细粒度深度交互，弥补 ITC 仅做全局对齐的不足，是交叉注意力机制最核心的应用场景之一。

### 2. 定量计算逻辑

1. 样本构建：对批次内的N个匹配图文对（正样本，标签 = 1），通过随机替换图像 / 文本，构建N个不匹配的图文对（负样本，标签 = 0）；
2. 跨模态交互：将图文对输入交叉注意力层，文本特征作为 Q，图像特征作为 K/V，得到融合了双向信息的跨模态特征hit；
3. 二分类预测：用一个线性分类头，将跨模态特征映射为 0~1 的匹配概率y^=Sigmoid(Linear(hit))；
4. 损失计算：用二元交叉熵损失（BCE）计算 ITM 损失：LITM=−2N1∑i=12N[yilogy^i+(1−yi)log(1−y^i)]其中yi为样本的真实标签（1 = 匹配，0 = 不匹配）。

### 3. 核心作用与特点

- 核心作用：学习细粒度的图文对应关系，让模型理解图像局部区域和文本词汇的对应关系，大幅提升模型在 VQA、图文推理、细粒度识别等下游任务的性能；
- 特点：必须依赖交叉注意力实现图文的深度交互，是交叉注意力在多模态预训练中的核心落地场景；
- 局限性：单塔融合结构，计算量比 ITC 更高，通常在 ITC 全局对齐的基础上做精调。

## （三）ITC 与 ITM 的核心对比与配合使用

### 1. 核心差异对比表

表格







|     维度     |       ITC 图文对比损失       |         ITM 图文匹配损失         |
| :----------: | :--------------------------: | :------------------------------: |
|   核心目标   |       全局语义级粗对齐       |        细粒度交互级精对齐        |
|   核心依赖   | 双塔独立编码器，无交叉注意力 | 单塔融合结构，必须依赖交叉注意力 |
|   损失类型   |       InfoNCE 对比损失       |        二元交叉熵分类损失        |
|   计算效率   |     高，适合大规模预训练     |        较低，适合精调优化        |
| 核心适配任务 |     图文检索、跨模态匹配     |    VQA、图文推理、细粒度理解     |

### 2. 标准配合范式

工业界主流的多模态预训练，均采用「ITC 先全局对齐，ITM 再细粒度优化」的两阶段范式，同时搭配 MLM（掩码语言模型）损失，形成经典的三损失预训练框架：

1. 第一阶段：用大规模无标注图文对，以 ITC 损失为主，快速学习全局的图文语义关联，完成基础对齐；
2. 第二阶段：用高质量图文对，以 ITM+MLM 损失为主，通过交叉注意力做细粒度的图文交互优化，提升模型的下游任务能力。

Qwen-VL、BLIP 等主流模型，均在预训练阶段沿用了这套 ITC+ITM 的组合范式。
